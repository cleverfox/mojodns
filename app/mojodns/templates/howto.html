API tokens

Most automation around mojodns — listing your zones, issuing TLS certificates, dynamic DNS — is authenticated with an API token. A token acts on your behalf: it can only reach zones you own or edit. Pass it either as the X-API-Key HTTP header or as a ?token=… query parameter.

TLS certificates (acme.sh / Let's Encrypt)

The panel is compatible with the stock dns_pdns module of acme.sh, so DNS-01 validation (including wildcard certificates) works without any custom plugins. Your token may only create and remove TXT records, in your zones only.

export PDNS_Url="{{ base }}"
export PDNS_ServerId="localhost"
export PDNS_Token="<token>"
export PDNS_Ttl=60

acme.sh --issue --dns dns_pdns -d example.com -d '*.example.com'

Validation records propagate to the public nameservers within a couple of seconds, so no extra --dnssleep is normally needed.

Dynamic DNS

Hosts with changing addresses can keep an A/AAAA record up to date. The endpoint is idempotent — if the address didn't change, nothing is rewritten — so calling it from cron every few minutes is fine.

raw API

# set/update a record (omit &ip=… to use the source address of the request)
curl -X POST -H "X-API-Key: <token>" \
  "{{ base }}/api/v1/ddns?name=home.example.com&type=A&ip=203.0.113.7&ttl=300"

# remove it
curl -X DELETE -H "X-API-Key: <token>" \
  "{{ base }}/api/v1/ddns?name=home.example.com&type=AAAA"

ready-made client (bash / FreeBSD sh)

Download mojodns-ddns.sh and mojodns-ddns.conf.example, then:

fetch {{ base }}/static/client/mojodns-ddns.sh   # or curl -O / wget
install -m 755 mojodns-ddns.sh /usr/local/bin/
curl -o /usr/local/etc/mojodns-ddns.conf {{ base }}/static/client/mojodns-ddns.conf.example
vi /usr/local/etc/mojodns-ddns.conf              # set PANEL_URL, TOKEN, RECORD

# every 5 minutes from cron:
*/5 * * * * /usr/local/bin/mojodns-ddns.sh

The client discovers the host's public IPv4/IPv6 with curl -4 / curl -6 and updates the records. Behaviour is controlled by:

UPDATE_A / UPDATE_AAAA which address families to manage (true/false)
DEL_STALE_A / DEL_STALE_AAAA what to do when that family has no connectivity: true — delete the now-stale record from DNS; false — leave it untouched (default)
IP4_URL / IP6_URL where to learn the public address (default https://ifconfig.me); set to auto to skip discovery and let the panel use the source address of the update request
TTL / CURL_TIMEOUT / QUIET record TTL (default 300), curl timeout, suppress normal output for cron

A safety detail: if the IPv6 path answers with an IPv4 address (NAT64 networks), the client treats it as "no IPv6" instead of publishing a broken AAAA record.
